# Yandex Agent

Короткая памятка по проекту.

## Что было сделано

Сначала я сделал обычный вход через Яндекс ID, чтобы пользователь мог зайти в интерфейс и получить свою сессию. Потом уже по ходу работы стало понятно, что для реального подключения Яндекс.Директа этого мало.

Я отдельно добавил backend-коннектор для Direct API, вынес работу с токенами на сервер, сделал подключение через verification code и перевел агент в безопасный режим рекомендаций. Сейчас агент тянет реальные данные, но не меняет боевые кампании автоматически.

Отдельно были добавлены тесты и улучшена логика рекомендаций. Теперь агент умеет давать советы даже на слабых данных из песочницы, а не только на полном наборе метрик.

## Как сейчас работает вход

Обычный вход на сайт идет через Яндекс ID. Это нужно для самого интерфейса, чтобы пользователь вошел в систему.

Подключение Директа идет отдельно. Для этого на backend настроено второе приложение с доступом к `direct:api`. Пользователь нажимает кнопку подключения, получает verification code в окне Яндекса и вставляет его обратно в приложение. После этого backend меняет код на токен и уже с этим токеном ходит в Direct API.

## Как подключить Директ

1. Открыть сайт.
2. Войти через Яндекс.
3. Нажать `Подключить Директ`.
4. В открывшемся окне Яндекса разрешить доступ.
5. Скопировать verification code.
6. Вернуться в приложение.
7. Вставить code в поле подключения.
8. Дождаться подтверждения подключения.

Если code вставлен успешно, дальше агент может получать кампании и статистику через backend-коннектор.

## Что важно сейчас

Сейчас на Render активирован режим песочницы. Это значит, что проект подключен к тестовой среде Яндекс.Директа и работает на sandbox API. Такой режим подходит для демонстрации, проверки логики и защиты проекта без риска для реальной рекламы.

Если нужно перейти на боевые данные, позже можно будет просто переключить backend с sandbox на production и заново подключить Direct уже с боевым доступом.

## Что умеет агент

- подключаться к Яндекс.Директу через backend-коннектор
- получать кампании и статистику
- считать базовые KPI: CPC, CPM, CPO, CR, ROAS
- строить рекомендации по эффективности
- работать в read-only режиме
- не вносить автоматические изменения в рекламу
- показывать лог цикла анализа
- выдавать диагностические рекомендации даже при слабом объеме данных

Сейчас логика сделана так, чтобы агент был полезен и на песочнице. Если кампаний мало или данных еще недостаточно, он пишет не просто что все нормально, а подсказывает, чего не хватает для осмысленной оптимизации.

## Что было улучшено в последней версии

Добавлены тесты на рекомендации и на backend.

Сейчас проект проходит:

- frontend / backend тесты
- сценарии подключения Direct
- базовую сборку production-версии

Также была улучшена логика рекомендаций:

- если кампаний нет
- если в портфеле только одна кампания
- если нет показов
- если есть показы, но нет кликов
- если кликов мало
- если есть клики и расход, но нет конверсий

## Как тестировать в песочнице

Лучше всего создать не одну, а 2-3 тестовые кампании с разным поведением. Тогда агенту будет легче сравнивать их между собой и давать более интересные советы.

Хороший тестовый сценарий такой:

- одна кампания почти без трафика
- одна с кликами, но без конверсий
- одна с более стабильной историей

Тогда рекомендации становятся заметно полезнее.

## Что можно развивать дальше

Сейчас агент работает по правилам и аналитике на метриках. Следующий шаг можно делать уже в сторону ML и LLM.

ML можно подключить для:

- поиска аномалий по истории кампаний
- прогноза расхода и кликов
- оценки вероятности просадки кампании
- сегментации кампаний по качеству трафика

LLM можно подключить для:

- объяснения рекомендаций простым языком
- генерации следующего шага для маркетолога
- разбора проблемной кампании в формате мини-отчета
- подготовки текстового summary по аккаунту
- ответа на вопросы пользователя прямо внутри агента

То есть дальше агент можно развивать не только как dashboard с правилами, а как полноценного ассистента по рекламе.
